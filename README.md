# Algorithm Examples for SEU

一个面向算法课程复习、代码理解和基础面试准备的 Python 算法样例仓库。

![Python](https://img.shields.io/badge/Python-3.10%2B-3776AB?style=flat-square)
![Tests](https://img.shields.io/badge/tests-18%20passing-brightgreen?style=flat-square)
![Docs](https://img.shields.io/badge/docs-Chinese-red?style=flat-square)
![Visualizer](https://img.shields.io/badge/visualizer-local%20browser-orange?style=flat-square)

不是竞赛模板库，也不是追求极限性能的算法框架。  
它更像一个认真整理过的学习项目：能直接运行、能看中文解释、能跑测试、还能在浏览器里看回溯算法的动画过程。

</div>

## 这是什么

这个项目主要解决三个常见场景：

- 课程复习时，想快速回顾算法的目的、核心思路、状态定义和复杂度。
- 看代码时，希望示例不要过度抽象，最好能直接运行并对照输出理解。
- 面试或考试前，需要一个结构清晰、分类明确、带测试保护的算法样例集合。

仓库当前覆盖的内容包括：

- 排序与查找
- 动态规划
- 贪心与图算法
- 分治与数值算法
- 回溯与搜索
- NP 完全相关理论辅助示例

同时还提供两种查看方式：

- `cli.py` / `demo.py`：命令行直接跑示例
- `visualize.py`：浏览器本地动画演示，适合观察回溯搜索过程

## 适合谁

这个仓库比较适合下面几类读者：

- 正在上算法课，想找一份中文、可运行、可复习的样例项目
- 准备期末考试，希望把“算法思想 + 复杂度 + 代码实现”串起来
- 刚开始系统看算法，想先从可读性强的实现入手
- 想在面试前快速复盘经典算法，但又不想只背结论

如果你要的是这些东西，它会比较合适：

- 讲清楚思路
- 示例尽量短而完整
- 注释和输出更偏教学
- 保留基础测试，方便修改后回归验证

如果你要的是这些东西，它就不是最优选择：

- 大规模数据下的性能优化
- 算法竞赛模板化极致写法
- 覆盖特别全面的题库平台
- 工业级图算法/数值计算库

## 项目亮点

### 1. 面向学习，而不是只给答案

代码实现尽量保持“读得懂”。  
很多算法都会保留中文说明，重点放在：

- 这个算法在解决什么问题
- 它为什么这样设计
- 关键状态或贪心选择是什么
- 时间复杂度和空间复杂度怎么分析

### 2. 既能跑代码，也能看过程

除了普通命令行输出外，项目还提供本地浏览器可视化入口：

- `N 皇后`
- `子集和`
- `全排列`

这些动画不是花哨展示，而是把“搜索、剪枝、回退、恢复状态”的过程拆成了真实步骤。  
你可以暂停、拖动进度，也可以用自定义输入重新生成动画。

### 3. 结构清晰，适合按主题复习

算法模块按主题拆分，避免所有内容堆到一个文件里。  
你可以按“排序 -> DP -> 图 -> 回溯 -> 分治 -> 理论”这样逐步学习，也可以只针对某一类题型集中复习。

### 4. 有基本测试，不是一次性笔记

这个仓库不是“写完就不管”的课堂草稿。  
目前已经包含基础回归测试，覆盖：

- 主要算法输出是否正确
- 边界输入是否得到合理处理
- CLI 主题注册是否正常
- 可视化接口和数据结构是否可用

## 你可以怎么用这个仓库

### 如果你是考前复习

建议这样走：

1. 先用 `python cli.py --list` 看有哪些主题
2. 从 `sorting`、`dp`、`graph` 开始
3. 每看一个算法，都自己复述一遍“问题、思路、复杂度、适用前提”
4. 最后结合 [docs/exam_quick_review.md](docs/exam_quick_review.md) 做速记

### 如果你是平时学算法

建议一边看源码，一边跑示例：

1. 用 `python demo.py` 先看整体输出
2. 打开对应算法文件，看中文注释和返回结构
3. 用测试确认你修改后的代码没有退化

### 如果你想演示给别人看

直接用：

```bash
python visualize.py
```

浏览器会打开本地页面，你可以切换动画、播放步骤、修改输入参数。  
这个模式很适合讲解“回溯到底怎么回去的”“为什么这里会剪枝”这种问题。

## 快速开始

### 环境要求

- Python `3.10+`
- 无必须安装的第三方运行依赖

### 获取项目

```bash
git clone git@github.com:zelo789/Algorithm-Examples-for-SEU.git
cd Algorithm-Examples-for-SEU
```

### 运行全部示例

```bash
python demo.py
```

### 按主题运行

```bash
python cli.py --list
python cli.py sorting
python cli.py dp
python cli.py graph
python cli.py divide
python cli.py theory
python cli.py backtracking
```

### 打开浏览器动画演示

```bash
python visualize.py
```

你也可以指定端口或只导出数据：

```bash
python visualize.py --port 8765
python visualize.py --dump-json
```

### 运行测试

```bash
python -m unittest
python -m compileall .
```

## 学习路线建议

如果你是第一次系统地过这份仓库，我建议按下面顺序：

```text
排序与查找
  -> 动态规划
  -> 贪心与图算法
  -> 回溯与搜索
  -> 分治与数值算法
  -> NP 完全相关理论
```

原因大致是：

- 排序、查找最容易建立“算法分析”的基本感觉
- 动态规划是考试和面试高频重点
- 贪心和图算法经常一起出现，但前提和适用条件必须分清
- 回溯算法非常适合理解“搜索树”和“状态恢复”
- 分治与 FFT / Strassen 更适合在基础稳了之后再看
- NP 完全理论更偏概念理解和答题表述

## 算法地图

### 第一层：最值得先掌握的内容

| 模块 | 典型算法 | 为什么建议优先看 |
| --- | --- | --- |
| 排序与查找 | 快速排序、归并排序、二分查找 | 最基础，也最适合训练复杂度分析 |
| 动态规划 | 0/1 背包、LCS、矩阵连乘 | 考试和面试都很常见 |
| 图算法 | Kruskal、Dijkstra | 容易出综合题，也容易混淆概念 |

### 第二层：非常适合理解算法思想的内容

| 模块 | 典型算法 | 重点关注 |
| --- | --- | --- |
| 回溯与搜索 | N 皇后、子集和、全排列 | 搜索树、剪枝、恢复状态 |
| 分治 | 最近点对、Strassen、FFT | 分解、递归、合并与递推分析 |
| 概率与随机 | 蒙特卡洛、概率 DP | 概率状态与随机化思想 |

### 第三层：偏理论或理解型的内容

| 主题 | 建议掌握到什么程度 |
| --- | --- |
| NP / NP-hard / NP-complete | 能说清概念差异 |

## 可视化说明

浏览器动画目前主要覆盖回溯算法，因为这一类算法特别适合“按步骤看过程”。

现在支持：

- `N 皇后`
- `子集和`
- `全排列`

页面里可以做这些事：

- 播放 / 暂停动画
- 拖动进度条定位到某一步
- 调整播放速度
- 修改输入并重新生成动画

当前对输入规模做了适度限制，原因不是功能做不到，而是为了保证本地页面仍然流畅、步骤数量仍然可观察。

## 这个仓库的边界

为了避免 README 看起来很“满”，但实际体验不匹配，这里把边界说清楚：

- 这里的实现以教学可读性优先，不以极致性能优先
- 很多算法选择了更容易理解的写法，而不是最短或最硬核的竞赛写法
- 可视化当前只覆盖回溯类算法，不代表其它算法不重要
- 测试已经能提供学习项目级别的保护，但它不是在线评测系统
- 部分理论问题的实现是“小规模可运行辅助”，重点在理解，不在求解大实例

## 考试和面试里怎么用它

你可以把这个仓库当成一个“把抽象算法问题落到代码上的中间层”。

复习时重点看：

- 状态定义怎么写
- 贪心选择为什么成立
- 图算法的适用前提是什么
- 回溯算法如何维护路径和恢复状态
- 每类算法的复杂度为什么是这个量级

答题时重点借鉴：

- 动态规划的状态描述方式
- 分治算法的分解-求解-合并结构
- 贪心算法的表述顺序
- 图算法中 MST 和最短路的区别

如果只是想快速过一遍概念，可以直接读 [docs/exam_quick_review.md](docs/exam_quick_review.md)。
