# 算法样例平衡扩展与可视化设计

## 目标

在保持项目轻量、复习友好的前提下，同时完成三件事：

1. 扩展一个常见且缺失的算法专题：回溯与搜索。
2. 优化命令行与示例入口的组织方式，减少重复维护。
3. 增加一个零依赖的本地网页可视化入口，用动画方式演示算法步骤。

## 范围

### 新增算法专题

新增 `algorithms/backtracking_search.py`，首批包含：

- `solve_n_queens`
- `subset_sum_exists`
- `generate_permutations`

这些算法分别覆盖约束搜索、剪枝判断和枚举生成，适合作为课程复习与动画演示样例。

### 统一示例注册

新增一个共享注册表模块，集中描述主题名、中文简介和示例运行函数。`cli.py` 与 `demo.py` 改为复用同一份元数据，避免双份维护。

### 可视化演示

新增一个本地网页演示入口，使用 Python 标准库提供 HTTP 服务，浏览器端使用原生 HTML/CSS/JS 播放动画，不引入 Flask、FastAPI 或前端构建工具。

首批可视化对象为：

- N 皇后放置与回溯过程
- 子集和搜索与回退过程
- 全排列构造过程

## 结构设计

### 算法层

保留现有“算法实现优先”的结构，新模块继续使用清晰注释和简单返回值。对于需要回溯路径的算法，返回结果对象而不是裸布尔值，便于测试和展示。

### 展示层

共享注册表负责“运行什么示例”，而不是承担算法逻辑。CLI、demo 和可视化页面都从注册表读取主题信息，但可视化动画数据由独立的 trace 构造模块负责。

### 可视化层

可视化数据使用可序列化的字典结构，包含：

- 算法标识与标题
- 示例输入
- 步骤列表
- 每一步对应的状态快照和说明文本

浏览器端负责播放、暂停、拖动进度和渲染当前状态，不把算法逻辑写进前端。

## 测试策略

1. 为回溯算法补正确性与边界测试。
2. 为共享注册表补基本集成测试，验证 CLI 可列出与运行新主题。
3. 为可视化 trace 数据补结构测试，确保页面消费的数据稳定可用。

## 非目标

- 不引入第三方 Web 框架或前端打包工具。
- 不把所有现有算法都改造成可视化版本。
- 不做复杂交互编辑器或在线代码执行环境。
