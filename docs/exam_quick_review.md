# 算法考试速记

## 分治算法

适用特征：

- 原问题可以拆成若干个规模更小、结构相同的子问题。
- 子问题之间相对独立。
- 子问题答案可以合并成原问题答案。

答题模板：

1. 分解：把规模为 n 的问题拆成子问题。
2. 解决：递归求解子问题。
3. 合并：把子问题结果合成最终答案。
4. 复杂度：写递推式，再用主定理或展开法分析。

典型例子：

- 归并排序：`T(n)=2T(n/2)+O(n)`，所以 `O(n log n)`。
- 快速排序：平均 `T(n)=2T(n/2)+O(n)`，所以平均 `O(n log n)`；最坏划分成 `0` 和 `n-1`，所以 `O(n^2)`。
- 最近点对：左右递归 + 中线条带检查，`O(n log n)`。
- Strassen：`T(n)=7T(n/2)+O(n^2)`，所以 `O(n^log2(7))`。

## 动态规划

适用特征：

- 最优子结构：大问题最优解包含小问题最优解。
- 重叠子问题：递归会反复计算相同小问题。

答题模板：

1. 状态定义：`dp[i][j]` 表示什么。
2. 状态转移：当前状态由哪些更小状态得到。
3. 初始化：边界状态是什么。
4. 遍历顺序：保证转移依赖已经算过。
5. 答案位置：最终答案在 `dp` 的哪个位置。
6. 复杂度：状态数乘每个状态转移代价。

常见状态：

- 0/1 背包：`dp[i][c]` 表示前 i 件物品、容量 c 的最大价值。
- LCS：`dp[i][j]` 表示两个字符串前缀的最长公共子序列长度。
- 矩阵连乘：`dp[i][j]` 表示矩阵 Ai 到 Aj 的最小乘法次数。
- 概率 DP：`dp[i][j]` 表示当前比分下最终获胜概率。

## 贪心算法

适用特征：

- 每一步做局部最优选择。
- 局部最优选择能导向全局最优。

证明方式：

- 交换论证：说明任意最优解都能换成包含当前贪心选择的最优解。
- 归纳证明：做完当前选择后，剩余部分仍是同类最优子问题。

典型例子：

- 活动选择：优先选择结束最早的活动。
- 哈夫曼编码：每次合并频率最小的两棵树。
- Kruskal：每次选择不成环的最小边。
- Dijkstra：每次确定当前距离最小的未确定点，但要求边权非负。

## 图算法

最小生成树和最短路不要混：

- 最小生成树：让所有点连通，边权总和最小。
- 最短路：从一个源点到其他点的路径距离最小。

Kruskal：

- 输入更像“边列表”。
- 先按边权排序。
- 用并查集判断是否成环。
- 复杂度 `O(E log E)`。

Dijkstra：

- 输入更像“邻接表”。
- 用优先队列反复取当前最短距离点。
- 不能处理负边权。
- 复杂度 `O((V+E) log V)`。

## NP 完全问题

核心概念：

- P：能在多项式时间内求解。
- NP：给出候选答案后，能在多项式时间内验证。
- NP-hard：至少和 NP 中所有问题一样难。
- NP-complete：既属于 NP，又是 NP-hard。

考试常见表述：

- SAT 是第一个被证明的 NP 完全问题。
- 3-SAT 是 SAT 的特殊形式，但仍是 NP 完全问题。
- TSP 判定版是 NP 完全问题。
- TSP 优化版通常说是 NP-hard。

## 高频易错点

- 二分查找的循环条件通常是 `left <= right`，更新时要排除 `mid`。
- 0/1 背包每件物品只能选一次，完全背包才允许重复选。
- Dijkstra 遇到负边权不适用。
- Kruskal 要求图连通，否则没有生成树。
- LCS 是子序列，不是子串；子序列可以不连续。
- FFT 输入长度通常补齐到 2 的幂。
- TSP 动态规划仍是指数复杂度，不是多项式算法。
